# 搜索词设计指南

## 核心原则

**不用身份标签，用场景+情绪/状态词**

| 错误示例 | 正确示例 |
|---------|---------|
| "新手妈妈" | "带娃 崩溃" |
| "育儿知识" | "半夜 宝宝 怎么办" |
| "睡眠训练" | "落地醒 解决不了" |
| "短视频" | "刷视频 停不下来 焦虑" |
| "追剧" | "熬夜追剧 停不下来" |

## 关键词设计公式

```
场景词 + 情绪词/状态词 + 动作词
```

## 关键词类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **场景词** | 用户在什么场景 | 半夜、凌晨、通勤、睡前、碎片时间、吃饭时 |
| **情绪词** | 用户什么情绪 | 崩溃、焦虑、慌、累、烦、后悔、担心、停不下来 |
| **动作词** | 用户在做什么/想做什么 | 怎么办、怎么解决、求助、查、搜、问、刷 |

## 第一阶段搜索词设计（泛化发现）

### 目标
覆盖"做什么→遇到什么→情绪如何→怎么办"完整链路，发现高价值场景。

### 设计检查清单

- [ ] 每个场景 5-10 个搜索词
- [ ] 避免同质化词（如"哄睡难"和"哄睡困难"只留一个）
- [ ] 去掉偏期望而非现状的词（如"想要自动记录"）
- [ ] 覆盖不同时间段（早晨/白天/晚上/深夜）
- [ ] 覆盖不同情绪状态（焦虑/无聊/疲惫/兴奋）

### 示例：育儿场景

| 场景 | 搜索词 |
|------|--------|
| 夜间育儿 | "半夜 宝宝 哭闹 怎么办"、"凌晨 喂奶 崩溃"、"夜醒 频繁 解决不了" |
| 喂养困难 | "辅食 不吃 焦虑"、"奶量 不够 担心"、"挑食 严重 怎么办" |
| 睡眠问题 | "落地醒 解决不了"、"哄睡 困难 累"、"自主入睡 训练 失败" |
| 健康担忧 | "发烧 慌 怎么办"、"疫苗 反应 担心"、"湿疹 反复 焦虑" |

### 示例：视频/内容消费场景

| 场景 | 搜索词 |
|------|--------|
| 沉迷问题 | "刷视频 停不下来 焦虑"、"熬夜 追剧 后悔"、"短视频 上瘾 怎么办" |
| 内容质量 | "推荐算法 太烂 烦"、"内容 同质化 无聊"、"信息茧房 焦虑" |
| 时间管理 | "想减少 刷手机 做不到"、"碎片时间 浪费 后悔"、"手机 使用时长 控制" |

## 第二阶段搜索词设计（聚焦验证）

### 目标
围绕第一阶段识别的 P0 机会点，深入验证用户现状。

### 设计框架

每个 P0 机会设计 4 类搜索词：

| 维度 | 说明 | 示例（睡眠场景） |
|------|------|----------------|
| **工具使用** | 用户用什么工具 | "睡眠记录APP"、"哄睡神器"、"白噪音 软件" |
| **痛点暴露** | 用户遇到什么困难 | "夜醒频繁 怎么办"、"落地醒 怎么解决"、"睡眠倒退期 崩溃" |
| **行为描述** | 用户当前行为 | "自主入睡训练"、"抱睡 改 拍睡"、"夜奶 断不了" |
| **期望表达** | 用户想要什么 | "希望宝宝睡整觉"、"求睡眠神器"、"想要 睡眠监测" |

### 设计检查清单

- [ ] 覆盖"用什么→查什么→遇到什么→期望什么"完整链路
- [ ] 避免同质化词
- [ ] 去掉偏期望而非现状的词
- [ ] 每个 P0 机会 5-10 个搜索词

## 搜索词评审流程

1. **初稿**：基于主题和场景快速列出搜索词
2. **自检**：对照检查清单逐项检查
3. **交叉验证**：用搜索词在平台试搜，确认结果质量
4. **优化**：根据试搜结果调整搜索词
5. **定稿**：输出结构化搜索词清单
