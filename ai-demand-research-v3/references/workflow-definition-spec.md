# 工作流定义HTML规范

## 设计目标

工作流定义HTML是**流程定义的展示工具**，不是进度跟踪工具。它的用途是：

1. **展示当前定义的整体工作流程** — 所有步骤、阶段、依赖关系
2. **支持迭代检查** — 每次交流后检查流程是否需要更新
3. **点击节点查看定义** — 每个步骤的输入规范、输出规范、关键检查点、代码模板位置
4. **版本对比** — 流程变更时高亮显示新增/修改/删除的步骤

## 不需要的功能

- ❌ 实时进度更新
- ❌ 节点状态动画（执行中/已完成）
- ❌ 自动刷新机制
- ❌ 文件系统监控

## 需要保留的功能

- ✅ 整体流程可视化（阶段→步骤的层次结构）
- ✅ 点击节点展开详情（定义、规范、检查点）
- ✅ 版本标记（v1/v2/v3，显示当前版本）
- ✅ 变更高亮（迭代时标记新增/修改的步骤）
- ✅ 导出功能（导出为图片或PDF，方便分享）

### 节点样式

| 状态 | 边框颜色 | 背景 | 说明 |
|------|---------|------|------|
| 正常 | `--border` | `--bg-card` | 标准步骤 |
| 新增 | `--new` | `rgba(16,185,129,0.1)` | 本次迭代新增 |
| 修改 | `--modified` | `rgba(245,158,11,0.1)` | 本次迭代修改 |
| 删除 | `--removed` | `rgba(239,68,68,0.1)` | 本次迭代删除（保留显示） |

## 数据格式

workflow-definition.html 内嵌 `WORKFLOW_DEFINITION` 对象：

```javascript
const WORKFLOW_DEFINITION = {
  version: "3.0",
  lastUpdated: "2026-05-14T16:30:00+08:00",
  changelog: [
    { version: "3.0", date: "2026-05-14", changes: ["新增三维标签化分析", "新增搜索词优化闭环"] },
    { version: "2.0", date: "2026-05-10", changes: ["新增三维标签化", "扩展多平台支持"] },
    { version: "1.0", date: "2026-05-01", changes: ["初始版本"] }
  ],
  phases: [
    {
      id: "phase0",
      name: "项目初始化",
      steps: [
        {
          id: "step0-1",
          name: "创建项目",
          definition: "创建项目目录结构，配置主题和平台",
          input: "调研主题、目标平台、初步方向",
          output: "项目目录、project.json、workflow-definition.html",
          checkpoints: ["目录结构完整", "project.json配置正确"],
          codeTemplate: "templates/init_project.py",
          references: ["references/project-structure.md"],
          status: "normal" // normal | new | modified | removed
        }
      ]
    }
  ]
};
```

## 迭代检查流程

每次与用户交流后，执行以下检查：

1. **是否需要新增步骤？**
   - 用户提出了新的分析维度
   - 发现了遗漏的环节

2. **是否需要修改步骤？**
   - 步骤定义不够清晰
   - 输入输出规范需要调整
   - 检查点需要补充

3. **是否需要删除步骤？**
   - 步骤冗余
   - 与其他步骤合并

4. **是否需要调整流程顺序？**
   - 步骤依赖关系变化
   - 并行化机会

如需更新：
1. 修改 `WORKFLOW_DEFINITION` 数据
2. 标记变更状态（new/modified/removed）
3. 更新 changelog
4. 保存新版本
5. 询问用户是否满意，满意后更新到 skill

## 与 skill 的同步

当用户对工作流定义满意时：
1. 将 `WORKFLOW_DEFINITION` 同步到 `SKILL.md` 的工作流章节
2. 更新 skill 版本号
3. 保存 changelog
