# AI产品需求调研报告改进指导

> 本指导基于2026-05-18 session中用户明确接纳、认可的改进要求整理而成。
> 用于指导后续session对调研报告进行改进。

---

## 一、数据筛选规范

### 1.1 必须排除的干扰主题

以下两个主题涉及太多干扰词，**必须从分析中完全排除**：

| 排除主题 | 原因 | 典型干扰词 |
|---------|------|-----------|
| **社交/分享** | "分享"、"转发"等词在UGC中高频出现，但多数表达的是社交行为而非功能需求 | 分享、转发、@好友、一起看 |
| **互动/问答** | "评论"、"弹幕"、"AI对话"等表达与"看视频时的AI功能"主题无关 | 弹幕、评论、AI聊天、对话 |

**执行要求**：
- 第一层筛选：排除明显无关内容（AI生图教程、编程工具、育儿内容、纯AI资讯）
- 第二层筛选：排除纯"分享转发"、"弹幕评论"、"一起看"、"AI对话"等无实质功能需求的表达
- 第三层筛选：语义分类时，若UGC仅表达社交/互动需求，标记为"其他"或不纳入相关统计

### 1.2 数据质量红线

- **所有数据必须真实**，禁止编造统计数字
- 若原始数据与主题相关性低于40%，需向用户确认是否继续
- 报告必须说明数据筛选的完整过程（原始→过滤→有效）

---

## 二、报告内容规范

### 2.1 完整逻辑链路（强制要求）

每个产品建议必须包含完整的推导链条，**缺一不可**：

```
UGC原始表达 → 主题分类 → 分布统计 → 交互量分析 → 情绪佐证 → 产品建议
```

**具体体现**：
1. **UGC观察**：引用具体UGC数量和典型原文
2. **细分拆解**：二层主题分布（如AI生成视频82条、自动剪辑37条）
3. **交互量验证**：平均点赞、最高点赞、平台分布
4. **情绪佐证**：主导情绪及数量（如"期待/渴望"19条）
5. **产品建议**：基于以上推导得出的具体功能建议

### 2.2 必须包含的数据维度

| 维度 | 要求 | 展示方式 |
|-----|------|---------|
| **一层主题分布** | 所有主题的数量和占比 | 柱状图 + 表格 |
| **二层主题分布** | Top10-15功能点的数量和占比 | 柱状图 |
| **情绪分布** | 6种情绪的数量和占比 | 柱状图 |
| **场景分布** | 6种场景的数量和占比 | 柱状图 |
| **平台分布** | 5个平台的数量和占比 | 柱状图 |
| **典型UGC** | 每个主题至少3条原文 | 卡片展示（含平台、点赞、标签） |
| **交互量分析** | 平均点赞、最高点赞 | 数据画像卡片 |

### 2.3 报告规模要求

- **HTML报告大小**：30-50KB（不含外部依赖）
- **有效UGC数量**：400-600条（原始采集后需确认）
- **相关UGC比例**：需明确标注（如185/523 = 35.4%）

## 三、数据可视化规范

### 3.1 技术方案

- **必须使用纯CSS实现柱状图**，禁止依赖外部CDN（如Chart.js、ECharts）
- 原因：确保离线环境可查看，避免网络问题导致图表无法加载

### 3.2 柱状图样式标准

```
结构：标签（左）| 柱状条（中，带数值）| 百分比（右）
颜色：每个维度使用固定配色方案
交互：无需复杂交互，静态展示即可
```

### 3.3 必须包含的图表

1. **平台分布图**：小红书（红）、抖音（黑）、快手（橙）、B站（蓝）、知乎（蓝）
2. **主题分布图**：10个主题，每个固定颜色
3. **情绪分布图**：6种情绪，区分正负情绪颜色
4. **场景分布图**：6种场景
5. **二层主题分布图**：Top10功能点

---

## 四、交互设计规范

### 4.1 主题详情折叠面板

- 每个一层主题使用可折叠面板（collapsible）
- 默认收起，点击展开
- 展开后显示：数据画像、典型UGC、二层分布、推导逻辑

### 4.2 UGC卡片规范

每张UGC卡片必须包含：
- 平台标签（带颜色）
- 点赞数
- 标题（加粗）
- 内容摘要（前200字）
- 二层主题标签（紫色）
- 情绪标签（黄色）

## 六、评分与公式规范

### 6.1 红线要求

> **分析中涉及评分、分数的，必须设计完整公式，必须在报告中给出详细公式，否则不可使用具体评分。**

### 6.2 公式展示要求

- 若使用优先级评分（P0/P1/P2），需说明评分维度（如UGC数量占比、交互量、情绪强度）
- 公式必须可验证、可追溯

## 七、报告结构模板

```
1. 执行摘要（4个核心统计卡片）
2. 数据概览与筛选逻辑
   - 筛选过程（三层过滤，带箭头流程）
   - 平台分布（柱状图）
   - 数据质量洞察（黄色提示框）
3. 一层主题深度分析
   - 主题分布总览（柱状图）
   - 逐个主题折叠面板（10个）
     - 数据画像（两栏：数量/交互量 + 平台/情绪/场景）
     - 典型UGC（3条卡片）
     - 二层主题分布（柱状图）
     - 推导逻辑（蓝色逻辑链）
4. 情绪与场景分析
   - 情绪分布（柱状图 + 洞察框）
   - 场景分布（柱状图 + 洞察框）
5. 二层主题深度分析
   - 二层主题分布Top10（柱状图 + 洞察框）
6. 产品建议推导
   - P0优先级（核心功能，3个，含完整推导链路）
   - P1优先级（差异化功能，3个）
   - P2优先级（长尾机会，4个）
7. 关键结论与行动建议（5条结论，每条含行动项）
```

## 九、用户核心诉求总结

1. **数据真实**：所有数字必须来自实际统计，禁止编造
2. **逻辑完整**：从UGC到产品建议的推导链条必须完整可追溯
3. **内容详细**：报告信息不能简略，需包含完整分析过程数据
4. **排除干扰**：社交/分享、互动/问答主题必须排除
5. **二层深度**：必须补充二层主题的详细分析
6. **可视化可用**：图表必须离线可查看，不依赖外部CDN
