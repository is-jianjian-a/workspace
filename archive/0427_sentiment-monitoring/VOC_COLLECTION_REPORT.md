## 一、汇报概述

### 1.1 什么是VOC收集？

VOC（Voice of Customer，用户声音）收集是指从社交媒体、论坛、评论区等公开渠道，采集用户对品牌/产品/服务的真实反馈、评价、吐槽、建议等原声数据。

### 1.2 我们的目标

建立一套自动化/半自动化的VOC收集系统，覆盖国内主流社交平台，为产品改进、舆情监控、用户洞察提供数据支撑。

### 1.3 核心挑战

国内主流平台（微博、抖音、小红书、B站等）均有**严格的反爬机制**，包括：
- IP风控（频率限制、封禁）
- 设备指纹检测（浏览器特征识别）
- 行为分析（鼠标轨迹、点击模式）
- Token签名验证（API请求加密签名）
- 验证码/滑块验证

**结论：没有平台提供官方内容搜索API，所有采集均需绕过反爬机制。**

## 三、各平台详细进展

### 3.1 微博（Weibo）— ⭐⭐⭐⭐ 最成熟

#### 当前状态：🟢 已可用

| 功能 | 状态 | 技术方案 | 实现细节 |
|------|------|---------|---------|
| 热搜榜单 | ✅ | 直接访问 `s.weibo.com/top/summary` | 无需登录，20条热榜 |
| 关键词搜索 | ✅ | 有头浏览器+Cookie | Playwright登录→保存cookie→无头搜索 |
| 微博内容提取 | ✅ | DOM解析 | 20条微博（用户原声+媒体内容） |
| 评论深挖 | ✅ | 有头浏览器+JS点击 | 进入单条微博→点击评论按钮→提取28条评论 |

#### 技术实现

```
工作流:
1. 有头浏览器打开 weibo.com
2. 用户扫码/密码登录
3. 保存Cookie到 weibo_cookies.json
4. 后续使用无头浏览器+Cookie执行搜索
5. 解析DOM提取微博内容
```

#### 实际产出示例

**搜索"网红白冰偷税"：**
- 央视新闻: "国家税务总局通报，追缴+罚款1891.24万元"
- 用户原声: "4000万粉丝，一堆豪车，被实锤偷税漏税，实在不应该啊"
- 用户原声: "流量从来不是法外之地，无论名气大小，都必须恪守法律底线"

**评论深挖：**
- 28条有效评论提取，包含用户态度、情感倾向

#### 限制与风险
- Cookie会过期（约1-2周），需定期重新登录
- 频繁搜索可能触发验证码
- 评论提取需逐条进入页面，效率较低

### 3.5 小红书（Xiaohongshu）— ⭐⭐ 低

#### 当前状态：🟡 部分可用（重大限制）

| 功能 | 状态 | 技术方案 | 实现细节 |
|------|------|---------|---------|
| 热榜 | ❌ | 小红书无热榜功能 | N/A |
| 关键词搜索-搜索页 | ✅ | Playwright CDP + 已登录Chrome | 可提取标题/作者/点赞数/封面图 |
| 关键词搜索-帖子详情 | ❌ | JS点击触发404 | 被反爬拦截 |
| 弹层内容提取 | ⚠️ | 需用户手动点击 | 脚本可检测弹层但同步困难 |
| 评论获取 | ❌ | 未成功 | 评论需滚动加载，提取逻辑复杂 |

#### 技术实现（已验证）

```
工作流:
1. 用户手动登录Chrome（扫码/密码）
2. Playwright通过CDP连接Chrome（端口9222）
3. 访问搜索页 URL
4. 提取搜索页数据（标题/作者/点赞数/封面图）
5. ❌ JS点击帖子 → 触发404（反爬拦截）
6. ⚠️ 用户手动点击帖子 → 弹层打开 → 脚本尝试提取弹层内容
```

#### 实际测试结果

| 测试项 | 结果 | 说明 |
|-------|------|------|
| 搜索页数据提取 | ✅ 成功 | 标题、作者、时间、点赞数、封面图 |
| JS点击帖子 | ❌ 失败 | 触发404，被反爬拦截 |
| 用户手动点击 | ✅ 成功 | 可正常打开帖子弹层 |
| 弹层内容提取 | ⚠️ 困难 | DOM结构复杂，内容加载延迟 |
| 评论获取 | ❌ 未成功 | 评论需滚动加载，提取逻辑复杂 |

#### 四种方法对比结论

| 方法 | 可行性 | 说明 |
|------|--------|------|
| 手动+脚本 | ⭐⭐ | 效率极低，仅适合少量采集 |
| 官方API | ❌ | 小红书不提供内容搜索API（仅电商API） |
| 第三方爬虫服务 | ⭐⭐⭐ | 技术本质相同，有基础设施优势但无突破 |
| SkillHub技能 | ⭐⭐⭐ | 大概率封装相同技术，无新技术 |

#### 核心结论

> **小红书是目前反爬最强的平台之一，没有可靠的全自动采集方案。**
>
> 已实现的搜索页数据提取是**当前技术条件下最优解**。

## 四、技术方案深度分析

### 4.1 已验证有效的技术栈

| 技术 | 用途 | 成功率 | 说明 |
|------|------|--------|------|
| Playwright有头浏览器 | 登录+Cookie保存 | 90% | 模拟真实用户操作 |
| Playwright无头浏览器+Cookie | 搜索执行 | 70-80% | 复用Cookie避免重复登录 |
| CDP连接本地Chrome | 小红书等特殊场景 | 60% | 需用户预登录 |
| DOM解析 | 数据提取 | 80% | 依赖页面结构稳定性 |
| JS点击+文本提取 | 评论深挖 | 50% | 需处理动态加载 |

### 4.2 反爬机制与应对

| 反爬维度 | 强度 | 我们的应对 | 效果 |
|---------|------|----------|------|
| IP风控 | 高 | 无代理IP（当前） | ⚠️ 需补充代理IP池 |
| 设备指纹 | 高 | Playwright基础伪装 | ⚠️ 部分有效 |
| 行为分析 | 中 | 有头浏览器模拟真人 | ✅ 有效 |
| Token签名 | 极高 | 未逆向 | ❌ 无法解决 |
| 验证码 | 中 | 有头浏览器绕过 | ✅ 有效 |
| Cookie过期 | 中 | 定期重新登录 | ⚠️ 需人工配合 |

### 4.3 技术瓶颈

1. **小红书**：Token签名验证+点击来源检测，无法全自动
2. **抖音**：Cookie过期快+必须使用有头浏览器
3. **评论深挖**：所有平台均需逐条进入页面，效率低
4. **视频内容**：B站/抖音视频内容需语音识别，未实现
5. **IP风控**：当前无代理IP，高频操作易被封

## 八、风险与建议

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 平台反爬升级 | 现有方案失效 | 持续跟进，多方案备份 |
| Cookie频繁过期 | 需人工重新登录 | 建立提醒机制，考虑自动化登录 |
| IP被封 | 无法访问 | 引入代理IP池 |
| 页面结构变更 | DOM解析失效 | 建立监控，快速修复 |
| 法律合规 | 数据采集合法性 | 仅采集公开数据，遵守平台规则 |

### 8.2 建议

1. **优先落地微博+B站+头条**：这三个平台技术最成熟，可快速产出VOC数据
2. **小红书接受现状**：搜索页数据已是当前最优解，不投入过多资源突破
3. **评估付费数据源**：5118、新榜等第三方服务可能更稳定，评估ROI
4. **建立监控机制**：定时检查各平台可用性，及时发现反爬升级
5. **数据存储**：建立数据库积累历史数据，支持趋势分析
