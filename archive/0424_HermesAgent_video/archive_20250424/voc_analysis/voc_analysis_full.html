我采用了混合分析方法，结合：

  1. 关键词匹配法基于预定义主题词典进行内容匹配

# 统计每个主题的VOC数量 theme_counts = defaultdict(list) for voc in voc_entries: content = voc['content'].lower() for theme, keywords in themes.items(): if any(kw in content for kw in keywords): theme_counts[theme].append(voc) # 输出主题分布 print("主题分布（一条VOC可能属于多个主题）：") for theme, vocs in sorted(theme_counts.items(), key=lambda x: -len(x[1])): print(f" {theme:12s}: {len(vocs):4d} 条")

情感×主题交叉分析
分析各主题下的情感分布，提取典型负面评论
# 情感×主题交叉分析 for theme, vocs in sorted(theme_counts.items(), key=lambda x: -len(x[1]))[:8]: sentiments = Counter(v['sentiment'] for v in vocs) total = len(vocs) neg_pct = sentiments.get('negative', 0) / total * 100 if total > 0 else 0 pos_pct = sentiments.get('positive', 0) / total * 100 if total > 0 else 0 print(f" {theme} ({total}条):") print(f" 负面: {neg_pct:.1f}% | 正面: {pos_pct:.1f}% | 中性: {100-neg_pct-pos_pct:.1f}%") # 显示该主题的典型负面评论 neg_samples = [v for v in vocs if v['sentiment'] == 'negative'][:2] for v in neg_samples: print(f" 💢 {v['content'][:80]}...")

# 应用×主题分析 app_theme_matrix = defaultdict(lambda: defaultdict(int)) for voc in voc_entries: # 提取应用名（从context字段） app = "未知" for a in ['TikTok', 'YouTube', 'Netflix', 'Snapchat', 'Instagram', '腾讯视频', '抖音', '快手', 'B站']: if a in voc.get('context', ''): app = a break # 匹配主题 content = voc['content'].lower() for theme, keywords in themes.items(): if any(kw in content for kw in keywords): app_theme_matrix[app][theme] += 1 # 输出各应用的主要痛点 for app in ['TikTok', 'YouTube', 'Netflix', '抖音', '腾讯视频']: if app in app_theme_matrix: top_themes = sorted(app_theme_matrix[app].items(), key=lambda x: -x[1])[:3] print(f" {app}:") for theme, count in top_themes: print(f" - {theme}: {count}条")

方法论总结
本次分析采用的核心方法与工具链

采用的分析方法

方法 说明 工具
关键词匹配 预定义主题词典，匹配VOC内容 Python + 正则匹配
情感分类 四分类标签（已标注） 基于App Store星级推断
交叉分析 主题×情感、应用×主题 Python Counter + defaultdict
词频统计 高频关键词提取 Python string.count()
典型抽样 提取代表性VOC 条件筛选 + 截断显示

关键假设和局限

  1. 主题词典是预设的：基于我对视频AI领域的知识预设了12个主题，可能存在遗漏
  2. 情感标签是推断的：App Store评论的情感基于星级（1-2星=negative，4-5星=positive）
  3. 一条VOC可多主题：实际执行中允许一条VOC匹配多个主题
  4. 未使用NLP模型：没有使用BERT、GPT等模型进行语义分析，纯粹基于关键词匹配

可改进点

  1. 使用LLM进行语义聚类：可以调用GPT-4对VOC进行自动主题分类
  2. 情感分析模型：使用预训练的情感分析模型替代星级推断
  3. 时间序列分析：按时间维度分析痛点变化趋势
  4. 用户画像关联：结合应用类型、用户群体进行细分分析
