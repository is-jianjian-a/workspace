数据清洗代码
import sqlite3 import pandas as pd # 连接数据库 conn = sqlite3.connect('/path/to/sqlite_tables.db') df = pd.read_sql_query("SELECT * FROM xhs_note;", conn) # 特殊格式处理函数 def parse_count(value): if not value: return 0 value = str(value).strip().replace(',', '') if '万' in value: return int(float(value.replace('万+', '').replace('万', '')) * 10000) return int(value) # 应用转换 df['total_interaction'] = ( df['liked_count'].apply(parse_count) + df['collected_count'].apply(parse_count) + df['comment_count'].apply(parse_count) + df['share_count'].apply(parse_count) ) df['text'] = df['title'].fillna('') + ' ' + df['desc'].fillna('')

分析流程（7步 + 3维标签化）
1
数据加载与清洗
读取SQLite数据，处理特殊互动量格式，计算总互动量
输出: 660条结构化数据，包含total_interaction字段
2
主题聚类分析
构建12个主题词典，匹配内容主题，统计覆盖率与互动表现
输出: 主题分布表，识别供给缺口（低覆盖+高互动）
3
需求信号提取
构建5类信号词典（痛点/决策/求助/期望/行为），匹配并统计词频
输出: 信号覆盖率、高频词、平均互动
4
内容质量评估
基于营销信号和UGC信号，将内容分类为纯营销/纯UGC/混合/中性
输出: 4类内容分布，评估可信度
5
高价值内容识别
筛选互动量>10000且含痛点/期望信号的内容，按互动排序
输出: 112条高价值内容，提取具体案例
6
产品机会映射
基于供给缺口和需求强度，评估产品机会优先级（P0/P1/P2）
输出: 4个P0, 6个P1, 2个P2机会
7
三维标签化分析（新增）
使用情境、触发-退出机制、用户群体三个维度进行标签化
输出: 3个维度 × 多个子类别的标签分布
关键发现
使用情境洞察：
• 家庭场景（15.9%）和带娃时（12.8%）是主要使用情境
• 焦虑状态（10.7%）远高于无聊（0.8%），用户主动搜索而非被动消费
• 运动时（9.8%）和睡前（6.2%）是重要场景
触发-退出机制洞察：
• 社交需求（92.9%）是最大触发因素，但多为从众心理
• 算法陷阱（9.8%）是主要退出障碍
• 放松满足（10.8%）vs 空虚后悔（1.1%）— 正面情绪占主导
用户群体洞察：
• 孕期（33.6%）和备孕（17.3%）是主要群体
• 新生儿（8.9%）和幼儿期（8.6%）内容集中，小月龄（3.2%）供给不足
• 中度用户（15.8%）占主导，重度用户仅1.1%
