# Interview Techniques & Facilitation Guide

Advanced techniques for conducting effective user interviews and extracting deep insights.

## Probing Techniques

**The 5 Whys**: When a participant states a problem or preference, ask "why?" up to 5 times to reach root motivation.
- P: "I don't use that feature." → "Why not?"
- P: "It's too complicated." → "What specifically is complicated?"
- P: "Too many steps to get started." → "Which steps feel unnecessary?"
- Continue until underlying need is revealed.

**Critical Incident Technique**: Ask about specific recent events rather than general opinions.
- Good: "Tell me about the last time you exported a report. What happened?"
- Bad: "How do you feel about our export feature?"

**Think-Aloud Protocol**: Ask participant to verbalize thoughts while performing a task.
- "Please speak your thoughts out loud as you try to create a new project."
- Note: Pause and wait—silence often means confusion, not completion.

**Projection Techniques**: When direct answers are limited, use projection.
- "If you were explaining this to a new colleague, what would you tell them?"
- "Imagine this feature didn't exist—how would you work around it?"

## Facilitation Best Practices

**Before the interview**:
- Test recording equipment and internet connection
- Prepare 10-15 questions but expect to use only 6-8
- Review participant's usage data (if available) for context
- Send pre-read if covering complex concepts

**During the interview**:
- Start with warm-up questions to build rapport (2-3 min)
- Use participant's vocabulary, not internal jargon
- Follow the participant's lead—depart from script if they reveal unexpected insights
- Manage time: if running long, skip lower-priority questions rather than rush
- Capture exact quotes, not paraphrased summaries
- Note non-verbal cues: hesitation, excitement, frustration

**After the interview**:
- Complete notes within 30 minutes while memory is fresh
- Tag key quotes with metadata (user type, sentiment, topic)
- Share top 3 insights with stakeholders within 24 hours
- Debrief: What surprised you? What contradicted assumptions?

## Note-Taking Framework

Use the AEIOU framework during observation sessions:
- **Activities**: What actions is the user taking?
- **Environment**: What's the physical/digital context?
- **Interactions**: How does the user interact with tools/people?
- **Objects**: What artifacts, devices, or documents are involved?
- **Users**: Who else is involved, and what are their roles?

## Common Biases to Avoid

- **Confirmation bias**: Seeking information that confirms existing beliefs
- **Leading questions**: "Don't you think the interface is confusing?" → "How would you describe this interface?"
- **Social desirability**: Participants saying what they think you want to hear
- **Recency bias**: Over-weighting recent experiences
- **Power user bias**: Over-representing vocal power users vs. silent majority

Mitigation: Recruit diverse participants, use neutral question wording, validate with quantitative data, triangulate across multiple research methods.
