# User Journey Mapping Guide

Facilitation instructions and templates for building effective journey maps.

## Workshop Setup (90-120 minutes)

**Participants**: 4-8 people including product manager, designer, engineer, customer support rep, and data analyst. Add 1-2 actual users if possible.

**Materials**: Miro/FigJam board with journey map template, sticky notes (virtual or physical), timer.

**Pre-work**: Distribute recent user research, support ticket summaries, and analytics data 2 days before session.

## Journey Map Template

Build the map across these stages (adapt to product-specific flow):

| Stage | Awareness | Consideration | Purchase/Sign-up | Onboarding | Regular Use | Support | Renewal/Churn |
|-------|-----------|---------------|-----------------|------------|-------------|---------|---------------|
| Actions | List 3-5 | List 3-5 | List 3-5 | List 3-5 | List 3-5 | List 2-4 | List 2-4 |
| Touchpoints | Channels used | Channels used | Forms/pages | Tutorial, UI | Features used | Ticket, chat | Billing page |
| Thoughts | Quotes | Quotes | Quotes | Quotes | Quotes | Quotes | Quotes |
| Emotion | -5 to +5 | -5 to +5 | -5 to +5 | -5 to +5 | -5 to +5 | -5 to +5 | -5 to +5 |
| Pain Points | Top 3 | Top 3 | Top 3 | Top 3 | Top 3 | Top 3 | Top 3 |
| Opportunities | Ideas | Ideas | Ideas | Ideas | Ideas | Ideas | Ideas |

## Facilitation Script

### Opening (5 min)
"Today we're mapping the experience of [persona] as they [goal]. Our objective is to find the highest-impact opportunities for improvement. Everyone's voice matters — support, engineering, and design perspectives are equally valuable."

### Stage 1: Define the Persona (10 min)
- "Who is our primary user for this journey? Be specific — not 'users' but 'a marketing manager at a 50-person SaaS company'."
- "What's their primary goal with our product?"
- Write persona on board as reference throughout.

### Stage 2: Map Actions (20 min)
- Go stage by stage. For each: "What is [persona] doing at this stage?"
- Use verbs: "searches Google", "reads comparison article", "signs up with email", "completes onboarding checklist"
- Limit to 3-5 most common actions per stage

### Stage 3: Add Touchpoints (10 min)
- For each action: "Where does this happen? What channel or feature?"
- Note if touchpoint is digital, physical, or human-mediated

### Stage 4: Capture Thoughts (15 min)
- "What might [persona] be thinking at this moment?"
- Use real quotes from research when available
- Distinguish stated thoughts from inferred thoughts (mark inferred with "?")

### Stage 5: Plot Emotions (10 min)
- Each participant places an emotion dot (-5 to +5) for each stage
- Average the dots, draw the emotion line
- Highlight the biggest peaks and valleys

### Stage 6: Identify Pain Points (15 min)
- "Where does [persona] experience friction, confusion, or frustration?"
- Use red sticky notes for pain points
- Vote on top 3 pain points per stage (dot voting)

### Stage 7: Generate Opportunities (20 min)
- For each major pain point: "What might we do to fix this?"
- Use green sticky notes for opportunities
- No filtering at this stage — quantity over quality

### Closing (5 min)
- "What surprised you? What contradicted your assumptions?"
- "Which 2-3 opportunities should we investigate first?"
- Assign owners for follow-up validation

## Post-Workshop Actions

1. **Digitize** the journey map within 24 hours while memory is fresh
2. **Validate** with 3-5 actual users: "Does this match your experience?"
3. **Score opportunities** using the scenario scoring framework in SKILL.md
4. **Share** with stakeholders: 1-page summary + full map
5. **Schedule** quarterly review to keep journey map current

## Remote Facilitation Tips

- Use Miro timer for each section
- Use "raise hand" feature to manage speaking order
- Break into breakout rooms for opportunity generation (2-3 people per room), then reconvene
- Assign a dedicated note-taker who is not facilitating
- Record the session (with consent) for reference
