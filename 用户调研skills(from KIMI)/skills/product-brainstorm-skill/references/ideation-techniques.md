# Ideation Techniques Library

10 structured techniques for generating product ideas. Use based on problem type and team energy.

## Technique 1: SCAMPER (Structured Modification)

Best for: Improving existing products/features.

Process: Go through each letter systematically. Spend 3 minutes per letter.

| Letter | Prompt | Example Output |
|--------|--------|---------------|
| S | What can we Substitute? | Replace text input with voice input |
| C | What can we Combine? | Merge search + filtering into one interface |
| A | What can we Adapt? | Borrow TikTok's discovery feed for B2B content |
| M | What can we Modify/Magnify/Minify? | Reduce onboarding from 10 steps to 1 |
| P | What other uses exist? | Let users share their dashboards as templates |
| E | What can we Eliminate? | Remove account creation requirement entirely |
| R | What can we Reverse/Rearrange? | Let users set price, vendors bid |

## Technique 2: Six Thinking Hats (Multi-Perspective)

Best for: Evaluating complex decisions with multiple stakeholders.

Run each "hat" for 3-5 minutes. Capture output separately:

```
White (Facts): _______________________________________________
Red (Feelings): ______________________________________________
Black (Risks): _______________________________________________
Yellow (Benefits): ___________________________________________
Green (Alternatives): ________________________________________
Blue (Process): ______________________________________________
```

## Technique 3: Bad Ideas First

Best for: Overcoming creative blocks and lowering inhibitions.

Process:
1. Phase 1 (10 min): Generate the WORST possible ideas — illegal, expensive, absurd, offensive
2. Phase 2 (10 min): For each bad idea, ask "What makes this bad?" → invert to find good ideas
3. Phase 3 (10 min): Refine inverted ideas into viable concepts

Example: "Charge users $1000 per login" (bad) → "What if we inverted the pricing model?" → "Freemium with premium power-user features"

## Technique 4: Assumption Reversal

Best for: Radical innovation and challenging industry norms.

Process:
1. List 5-10 fundamental assumptions about the problem/industry
2. Reverse each assumption (state the opposite)
3. Ask "If the reversal were true, what product would exist?"

Example assumptions for food delivery:
- "Restaurants prepare food" → reversed: "Users cook, we deliver ingredients" (HelloFresh)
- "Delivery takes 30+ minutes" → reversed: "Food arrives in 10 minutes" (dark kitchens)
- "Users choose specific restaurants" → reversed: "Users choose cuisine, we pick the restaurant" (some AI apps)

## Technique 5: User Persona Swap

Best for: Finding accessibility and inclusivity opportunities.

Process: Redesign the experience as if the user were:
- A first-time user with zero domain knowledge
- A user with limited technical ability
- A user in a hurry (30 seconds only)
- A user with a disability (visual/motor/cognitive)
- A user in a distracting environment (noisy, mobile, one-handed)
- A user from a different culture or language

Each perspective reveals friction points invisible to the "average user" design.

## Technique 6: Time Travel

Best for: Understanding evolution and future-proofing.

Process:
1. **Past** (5 min): "How was this problem solved 10 years ago?" List manual, analog, or early-digital approaches
2. **Present** (5 min): "How is it solved today?" List current digital solutions
3. **Future** (10 min): "How will it be solved in 10 years?" Assume AI, ambient computing, neural interfaces, etc.
4. **Bridge**: "What steps connect present to future?" These are innovation opportunities

## Technique 7: Constraint Storming

Best for: Finding elegant solutions under limitations.

Process: Add artificial constraints that force creative solutions:
- "Design it for a feature phone with no internet"
- "Users can only interact via voice"
- "Must work without any backend infrastructure"
- "Must be explainable in one sentence"
- "Must require zero onboarding"
- "Must be usable by a 5-year-old and an 85-year-old"

Constraints breed creativity by eliminating obvious solutions.

## Technique 8: Jobs-to-be-Done Trigger Questions

Best for: Finding unmet user needs.

Ask about the user's "job" in different contexts:
- "When you're [situation A], what job are you hiring a product to do?"
- "What job are you struggling to get done because no good solution exists?"
- "What job did you stop trying to get done because it was too hard?"
- "What job do you reluctantly pay for because there's no better option?"
- "What job do you do poorly because you lack the right tool?"

Each question surfaces different opportunity types.

## Technique 9: Component Mapping

Best for: Platform and ecosystem opportunities.

Process:
1. Map all components of the current experience (data inputs, processing, outputs, user actions, integrations)
2. For each component, ask:
   - "Can this be unbundled into a standalone product?"
   - "Can an external provider do this better?"
   - "Can this be bundled with adjacent components for new value?"
   - "What if this component were the primary product?"

Example: Slack unbundled "team chat" from the full project management suite.

## Technique 10: What Would X Do?

Best for: Borrowing strategies from successful companies.

Process: Pick 3 companies known for innovation in different domains. Ask "How would [company] solve this problem?"

- "How would Apple solve this?" → Focus on simplicity, premium experience, ecosystem
- "How would Amazon solve this?" → Focus on speed, selection, customer obsession, data
- "How would Tesla solve this?" → Focus on vertical integration, bold bets, direct-to-consumer
- "How would Duolingo solve this?" → Focus on gamification, habit formation, freemium
- "How would Notion solve this?" → Focus on flexibility, community templates, bottom-up adoption

Map their approach patterns to your problem space.

## Technique Selection Guide

| Situation | Recommended Technique | Why |
|-----------|----------------------|-----|
| Improving existing feature | SCAMPER | Systematic modification |
| Team disagreement on direction | Six Thinking Hats | Structured debate |
| Creative block | Bad Ideas First | Lowers inhibition |
| Radical innovation needed | Assumption Reversal | Challenges fundamentals |
| Accessibility concerns | User Persona Swap | Multiple perspectives |
| Strategic long-term thinking | Time Travel | Future orientation |
| Resource-constrained | Constraint Storming | Elegance under limits |
| New market exploration | Jobs-to-be-Done Triggers | Need discovery |
| Platform strategy | Component Mapping | Architecture thinking |
| Stuck in conventional thinking | What Would X Do | Cross-industry inspiration |
