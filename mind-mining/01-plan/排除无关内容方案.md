# Mind-Mining 项目：排除无关内容方案

## 一、项目背景

- **项目名称**: mind-mining
- **数据来源**: 华为苹果性能对比项目原始数据库抓取数据，共1,923条帖子
- **目标**: 从1,923条原声中筛选出与华为/苹果手机真正相关的帖子，区分"相关内容"与"无关内容"，并识别"有讨论引子但无实质内容的帖子"建议深挖

## 二、数据现状

### 2.1 数据结构
- 原始JSON字段: `note_id`, `title_raw`, `title_cleaned`, `content_raw`, `content_cleaned`, `liked_count`, `comment_count`, `share_count`, `collected_count`, `source_keyword`, `tags`, `nickname`, `ip_location`, `note_url`
- 时间信息: 无直接`create_time`字段，但`note_id`前8位为16进制时间戳（秒级），可转换为发布时间
- 入库顺序: 以`note_id`字符串排序近似代表入库顺序

### 2.2 当前分类结果（基于之前935条对比类关键词）
| 类别 | 数量 | 占比 | 说明 |
|------|------|------|------|
| valid | 289 | 30.9% | 涉及性能/流畅度/稳定性/卡顿/丝滑维度 |
| unrelated | 594 | 63.5% | 涉及华为苹果但不讨论目标维度，或根本不涉及 |
| incomplete | 52 | 5.6% | 内容为空或过短，无法判断 |

### 2.3 无关内容的主要类型（基于594条unrelated分析）
1. **拍照/影像对比**（176条）：标题命中"对比"但实际讨论拍照、人像、live图
2. **仅讨论单一品牌**（146条）：78条仅华为，68条仅苹果
3. **平板/手表/非手机产品**（55条）：iPad vs MatePad、手表对比等
4. **价格/销量/市场新闻**（48条）：销量排名、降价新闻等
5. **信号/续航/充电/发热**（19条）：体验相关但非目标维度
6. **外观/设计/屏幕/颜色**（23条）：颜值、设计对比
7. **纯求助/无实质内容**（20条）：只有标题无正文
8. **其他品牌为主**（13条）：vivo/OPPO/小米等
9. **其他综合类**（77条）：双持分享、品牌情感等

## 三、相关性定义（本项目标准）

### 3.1 "相关"判定标准
与以下任一相关的帖子均算"相关"：
- **华为手机**（含Mate系列、Pura系列、nova系列、Pocket系列、折叠屏等）
- **鸿蒙手机**（含鸿蒙系统、HarmonyOS等）
- **苹果手机**（含iPhone全系列、iOS系统）

**判定方法**：帖子标题或正文中明确提及上述品牌/产品/系统之一，且内容围绕手机产品本身（排除平板、手表、耳机等配件，除非明确讨论手机性能/体验）。

### 3.2 "无关"判定标准
- 完全不涉及华为/苹果手机（含鸿蒙/iOS）
- 仅涉及华为/苹果的平板、手表、耳机、电脑等非手机产品
- 仅涉及其他品牌（vivo/OPPO/小米/三星/荣耀等）
- 纯营销广告、抽奖活动、与手机无关的生活分享

### 3.3 "建议深挖"判定标准（Discussion Prompt）
同时满足以下条件：
1. 标题或内容明确提及华为/苹果手机相关话题（有讨论引子）
2. 但正文内容**少于50字符**或**无实质展开**
3. 互动量（点赞+评论+收藏）**超过100**（说明引发了讨论，但原帖信息不足）

**典型例子**：
- "华为和苹果二选一你们选哪个" + 正文"只想自留一个，给点建议吧" + 评论105条
- "苹果这么难用为啥还那么多人买 想不通" + 正文极短 + 高互动

## 四、搜索有效性验证方法

### 4.1 方法概述
按搜索关键词的入库顺序排序，横轴为入库序号（时间近似），纵轴为1/0相关性，观察是否随入库时间往后，相关性概率越低。

### 4.2 具体步骤
1. **数据准备**：将每个关键词的帖子按`note_id`排序（近似入库顺序）
2. **滑动窗口**：设定窗口大小（如50条），步长（如25条），计算每个窗口的相关率
3. **线性回归**：对窗口中心位置与相关率做线性回归，判断趋势
4. **累积分析**：计算"搜索N条能获得多少有效内容"，找到边际效益拐点

### 4.3 当前发现（基于935条对比类数据）
- **趋势斜率**: -0.000116（微弱下降）
- **R²**: 0.076（解释力弱）
- **关键拐点**: 前200条新增valid率30-50%，200-400条骤降至15-25%，之后波动在20-35%
- **结论**: 未呈现典型的"衰减到0"模式，可能说明搜索算法持续返回混合质量结果

### 4.4 建议搜索条数
| 策略 | 条数/关键词 | 预期valid率 | 理由 |
|------|------------|------------|------|
| 保守 | 150-200 | 35-45% | 前200条效率最高 |
| 平衡 | 250-300 | 30-35% | 覆盖长尾，边际效益下降 |
| 穷尽 | 400+ | 25-30% | 当前数据无持续衰减证据 |

## 五、执行方案

### Phase 1: 数据迁移与整合
1. 将1,923条原始数据统一整合到`mind-mining/00-raw/`目录
2. 提取`note_id`时间戳，生成`publish_time`字段
3. 按`source_keyword`分组，建立数据索引

### Phase 2: 相关性标注（LLM批量分类）
1. **分类标准**：
   - `related`：与华为/苹果手机相关
   - `unrelated`：无关
   - `discussion_prompt`：有讨论引子但无实质内容（建议深挖）
2. **批次设计**：每批20-30条，并行处理
3. **输出要求**：每条标注分类+理由+置信度

### Phase 3: 搜索有效性验证
1. 按关键词分组，按`note_id`排序
2. 生成"入库顺序-相关性"散点图
3. 计算滑动窗口相关率与线性回归
4. 输出"搜索多少条足够"的建议

### Phase 4: 质量审计与报告
1. 抽样人工复核（每类10-20条）
2. 生成可交互HTML审核报告
3. 输出最终清洗后数据集

## 六、质量红线

1. **分类必须基于内容语义理解**，禁止纯关键词匹配
2. **"建议深挖"帖子必须同时满足三个条件**（有引子+无内容+高互动）
3. **搜索有效性分析必须使用真实数据**，禁止编造
4. **每个分类结果必须附理由**，便于人工复核
5. **时间戳转换必须验证**，抽样核对note_id时间与内容中的时间线索是否一致

## 七、待确认事项

1. **1,923条完整数据的位置**：当前仅看到935条对比类数据，单品牌关键词的约988条数据在哪个目录？
2. **"建议深挖"的互动量阈值**：当前设为100，是否需要调整？
3. **平板是否算"相关"**：如"华为MatePad vs iPad"是否纳入？当前方案排除，需确认。
4. **鸿蒙生态设备**：如鸿蒙车机、鸿蒙手表是否算"鸿蒙手机"？当前方案排除非手机。
