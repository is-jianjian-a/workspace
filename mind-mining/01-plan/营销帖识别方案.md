# 营销帖/人工帖识别方案

## 一、问题背景

在1,676条相关帖子中，需要区分：
- **真实UGC**：用户基于真实体验的发帖，含个人情感、使用细节
- **营销/引流帖**：以引流、带货、引战为目的，内容模板化、缺乏真实体验
- **讨论帖**：有讨论引子但内容较空，评论区才是主战场（已在另一流程处理）

## 二、识别难点

单一指标无法准确判断：
- 评论数>>点赞数：可能是引战帖，也可能是真实争议话题
- 分享率高：可能是干货被转发，也可能是营销裂变
- 内容短：可能是讨论帖，也可能是纯引流
- 需要**多维度组合评分 + 内容语义分析**

## 三、评分模型（0-100分）

### 3.1 内容维度（0-40分）

| 指标 | 条件 | 得分 | 说明 |
|------|------|------|------|
| 内容极短 | 正文<20字 | +10 | 缺乏实质内容 |
| 无第一人称 | 无"我/我的/入手/买了/用了"等 | +8 | 真实体验通常有第一人称 |
| 结构化编号 | 含[一R][二R]或①②或1️⃣2️⃣ | +10 | 像产品说明书/测评模板 |
| 纯参数罗列 | 含≥3个参数词（搭载/配备/像素/芯片等） | +8 | 缺乏主观感受 |
| 有详细体验 | 含≥3个体验词且正文>80字 | -15 | 真实使用痕迹，减分 |
| 有真实情感 | 含"后悔/失望/惊喜/无语/崩溃"等 | -10 | 真实情感表达，减分 |

### 3.2 互动维度（0-35分）

| 指标 | 条件 | 得分 | 说明 |
|------|------|------|------|
| 评论率异常 | 评论>3倍点赞 且 评论>50 | +15 | 引战/水军特征 |
| 分享率极高 | 分享>50%点赞 且 点赞>30 | +10 | 营销裂变特征 |
| 收藏率极低 | 收藏<3%点赞 且 点赞>50 | +5 | 干货帖通常收藏高 |
| 赞评比极高 | 点赞>10倍评论 且 点赞>50 | +5 | 可能买赞/低质内容 |

### 3.3 标题维度（0-25分）

| 指标 | 条件 | 得分 | 说明 |
|------|------|------|------|
| 标题党词汇 | 含"选哪个/怎么选/纠结/对比/伤害/详解/必看"等 | +8 | 引流标题特征 |
| 无情感表达 | 全文无"！/？/😂/💔/生气/喜欢"等情感符号 | +10 | 营销内容通常冷静客观 |

## 四、分级标准

| 等级 | 分数 | 数量(预估) | 处理建议 |
|------|------|-----------|---------|
| 🔴 高嫌疑 | ≥30分 | ~83条(5%) | 需人工复核，大概率排除 |
| 🟡 中嫌疑 | 20-29分 | ~197条(12%) | 建议抽样复核 |
| 🟢 低嫌疑 | <20分 | ~1396条(83%) | 保留进入下游分析 |

## 五、高嫌疑帖典型特征

### 5.1 纯引战型（评分40+）
- 内容=标题，或正文<10字
- 评论数>>点赞数（3-30倍）
- 无个人体验，纯抛问题引战
- 例："感觉华为比苹果好太多了"（336赞/1001评，内容仅标题）

### 5.2 参数罗列型（评分30-40）
- 结构化编号[一R][二R][三R]
- 纯参数对比，无个人使用感受
- 收藏/分享率高（被当购机参考转发）
- 例："华为Pura80系列参数对比！一图看懂怎么选"

### 5.3 标题党引流型（评分30-40）
- 标题含"必看/详解/推荐/怎么选"
- 正文极短或纯话题标签
- 赞评比异常（买赞或算法推荐）
- 例："iPhone这样设置手机秒变丝滑流畅"（12字正文，438赞）

## 六、与"讨论帖"的区分

| 特征 | 讨论帖 | 营销帖 |
|------|--------|--------|
| 内容长度 | 极短（<100字） | 可短可长 |
| 评论数 | 高（>300） | 可高可低 |
| 内容性质 | 有讨论引子（问句/争议点） | 有引流目的（对比/推荐/参数） |
| 第一人称 | 可有可无 | 通常无 |
| 情感表达 | 可有可无 | 通常无 |
| 结构化 | 无 | 常有[一R][二R]编号 |

**关键区分**：讨论帖是"抛出问题等评论"，营销帖是"输出内容引流转粉"

## 七、执行建议

### 7.1 自动化初筛
- 运行评分模型，给所有帖子打分
- 高嫌疑(≥30分)自动标记，进入人工复核

### 7.2 人工复核（高嫌疑帖）
- 打开原帖URL，观察：
  1. 账号历史发帖是否垂直
  2. 评论区是否模式化（水军痕迹）
  3. 内容是否真实可验证
- 复核后标记：确认营销 / 误判保留

### 7.3 输出
- 营销帖清单（note_id + 原因）
- 清洗后数据集（排除确认的营销帖）
- 人工复核记录（供后续优化模型）

## 八、数据需求

当前可用数据：
- ✅ 标题、正文、互动数据（赞/评/藏/转）
- ✅ 搜索关键词
- ❌ 账号信息（粉丝数、历史发帖、认证状态）
- ❌ 评论区内容
- ❌ 图片/视频内容

**如需更精准判断，建议补充**：
1. 账号粉丝数 + 历史发帖频率
2. 评论区抽样内容（检测水军模式）
3. 图片是否含商品链接/二维码

## 九、时间预估

| 步骤 | 时间 |
|------|------|
| 自动化评分 | 10分钟 |
| 高嫌疑帖人工复核（83条） | 2-3小时 |
| 中嫌疑帖抽样复核（50条） | 1小时 |
| 输出清洗后数据 | 10分钟 |
| **总计** | **约4小时** |

## 十、质量红线

1. **宁可错留，不可错杀**：误判营销帖=丢失真实UGC，影响下游分析
2. **必须有URL可复核**：每条高嫌疑帖都需提供原帖链接人工验证
3. **保留复核记录**：记录判断依据，便于后续模型迭代
4. **消费者视角优先**：营销帖判断标准与UGC需求研究目标一致
